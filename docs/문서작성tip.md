# 문서 작성 Tip (AI BOX)

AI BOX에 글을 올릴 때 참고하는 양식 안내입니다.
**이 파일은 앱에 노출되지 않습니다** — 앱은 `posts/<카테고리>/` 폴더 안의 글 파일만 불러오기 때문입니다.
(카테고리: 자유 · 정보 · 그림프롬프트 · 프롬프트)

## 어디에 쓰나 (깃허브·드라이브 모두 "파일 단위")

- **깃허브**: `posts/<카테고리>/` 폴더에 파일을 하나 만든다. **파일 하나 = 글 하나.**
  - 예) `posts/정보/GPT 소식 (2026-07-20).md` 를 만들면 → 정보 탭에 글이 올라간다.
  - 파일명이 곧 제목이 된다.
- **구글 드라이브**: 카테고리별 공유 폴더에 문서를 넣는다. 폴더 ID는 앱 설정(⚙)에 등록.

## 글 형식 — 한 파일 = 하나의 글

- **파일 하나(문서 하나)가 곧 글 하나**가 된다. 파일명이 제목이 된다: `제목 (2026-07-20).md`
- 파일 안에는 본문만 쓰면 된다.
- 끝의 `(YYYY-MM-DD)`는 날짜 표기 — 생략 가능(생략하면 파일 수정 시각으로 정렬).

### 긴 자료를 섹션으로 나눌 때

- 큰 자료를 소제목으로 나누고 싶으면 본문 안에서 `##`(또는 `#`·`###`)를 쓴다.
- 이 헤딩은 **글을 쪼개지 않고**, 상세 화면에서 **섹션 제목(소제목)** 으로만 표시된다.
- 즉 `## 결론`, `## 반복 생성 방식` 처럼 여러 섹션이 있어도 **여전히 글 하나(카드 하나)** 다.

```
큰 주제 한 편.

## 첫 번째 섹션
본문…

## 두 번째 섹션
본문…
```

## 사진

- `posts/<카테고리>/` 폴더에 이미지 파일(png·jpg·gif·webp 등)을 넣으면 사진 글이 된다.
- 목록에는 제목(파일명)만 보이고, 글을 누르면 사진이 뜬다. 사진을 탭하면 원본으로 확대.

## 본문에서 되는 것

- **굵게**는 `**굵게**` 로.
- 이미지 삽입: `![설명](https://…이미지주소)`
- 줄바꿈은 그대로 유지된다.
- 길이 제한 없음 — 수천 자 긴 글도 상세 화면에서 스크롤로 읽힌다.
- 코드블록(``` ``` ```)은 지원하지 않는다(그냥 글로 표시).

## 정렬 규칙 (참고)

글마다 하나의 시간값(제목 날짜가 있으면 그 날짜, 없으면 파일 시각)으로 환산해
**소스·날짜 유무와 무관하게 최신순**으로 정렬한다. 정확한 순서를 원하면 제목(파일명)에 날짜를 넣는다.
