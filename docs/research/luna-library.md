# 루나의 도서관 AI마을 — 자매 프로젝트에서 이식한 것 (2026-07-19)

> 루나의 첫 번째 마을. LLM 기반 "말의 창발" 실험으로, LUNA-WORLD와는 의도적으로 반대 극에 있다.
> 이 문서는 2026-07-19의 두 프로젝트 비교에서 도출해 **실제로 이식한 6개 부품**의 기록이다.
> 원 설계는 luna-x59q 저장소의 `docs/ai-village.md`(canonical) 참조.

## 무엇인가

Flask + Firestore 위에서 주민 9명이 LLM으로 살아가는 운영형 공동체. 관계를 일급 설계
대상으로 놓고 4요소(환경·상호작용·역사와 기록·소문)로 분해했으며, 핵심 루프는:

> 특정 장소에서 상호작용하고 → 인연이 생기며 → 기억으로 남고 → 소문이 퍼지며 → 다시 상호작용한다.

LUNA-WORLD가 원래 갖고 있던 것은 이 루프의 **앞 방향(사건 → 관계)**뿐이었다.
도서관 마을의 핵심 통찰은 **뒤 방향(관계 → 사건)** — "쌓인 인연은 다음 만남을 끌어당기는
중력이 되어야 한다"는 것이다.

## 이식하며 번역한 규칙 (LLM 문법 → 규칙 문법)

| 도서관 마을 (LLM) | LUNA-WORLD (규칙) |
|---|---|
| **약속** — 대화에서 방문·공동연구 약속이 창발, plan 힌트로 유도 | 좋은 대화에서 확률적으로 방문·공동작업 약속 객체 생성. 기한 창이 열리면 `chooseDestination`이 약속 장소로 끌어당기고, 둘이 실제로 같은 자리에 서면 이행·아니면 파기 |
| **쌍 중력** — 닫히지 않은 약속·의문이 재회를 끌어당기는 매턴 재계산 임시값 | `gravityList()` — 미이행 약속(+30), 풀지 못한 어긋남(+26), 둘 다 들은 미확인 소식(+10)의 합. 대화 확률과 자유시간 동선에 곱해지고, 매듭이 풀리면 스스로 사라진다 |
| **약속 파기의 파생** — 반복 파기→평판 소문, 마을 전체 누적→관습의 씨앗 (§2-Ⓐ) | 파기 2회→평판 소문(subject 있는 소식) 생성, 마을 파기 4회 누적→첫 관습 탄생(이후 약속 이행 확률 보정) |
| **기억-정합 소문** — 일치=증폭, 충돌=의심·역소문, 무경험=첫인상 | `giveNews()`가 듣는 이의 기억(meta.with/agent)과 대조해 확신도를 증폭·감쇠. 충돌 + 좋은 직접 경험 2개 이상이면 역소문(옹호 소식)을 실제로 생성 |
| **욕망 되먹임** — 성찰이 남긴 바람이 다음 계획의 선택압 | 기억 종류→욕구 테이블: 갈등→화해, 완성→자랑(광장 글로 풀림), 낡음→손질(광고 점수 +16), 저기분+사교→어울림(광장으로). 풀리면 사라지고, 안 풀리면 옅어진다 |
| **개인 거주지** (§2-Ⓔ) — 내면이 밖으로 드러나는 공간, 장소에 사건의 결 | 주민 8명 각자의 거처 건물. 밤 쉼과 방문 약속의 무대. `tend` 작업으로 굳은 기억(묶음)이 "방 안의 흔적"으로 남는다. 모든 건물에 `장소의 역사`(사건 로그)와 파생 분위기 텍스트 |
| **광장 글** — 오프라인 co-presence를 보완하는 온라인 전파 채널 | 광장에서 글을 쓰고, 나중에 온 주민이 시간차로 읽는다. 소식은 글을 타고도 퍼진다 (전파 경로에 기록) |
| **행사의 창발** (Smallville 발렌타인 파티 문법) | 주민이 모임을 발의 → 초대장이 기존 소식 전파망을 탄다 → 참석은 관계·사교성·기분이 결정. 3명 미만이면 주최자의 실패 기억이 남는다 — 그것도 이야기다 |

## 옮기지 않은 것 (의도적)

- **무수치 불변식** — 도서관 마을은 관계를 숫자로 두지 않고 텍스트로 창발시킨다.
  이는 LLM이 있어야 성립하는 원칙이다. LUNA-WORLD는 수치가 곧 엔진이므로 rel 수치를 유지하되,
  중력만은 도서관 문법대로 "저장하지 않고 매번 재계산하는 임시값"으로 옮겼다.
- **작가형 GM 금지의 엄격 적용** — 우리는 RimWorld식 사건 감독을 이미 채택했다 (관찰 재미가 목적이므로).
- **활성 3인 상한** — 비용 문제는 브라우저형에는 없다.

## 지킨 공통 원칙

**정보 비대칭 보존** — 모두가 모든 걸 알면 소문·오해·발견이 전부 죽는다. 소식은 여전히
사람(대화)과 글(게시판)을 타고만 퍼지고, 역소문·평판도 각자의 기억과의 대조에서만 나온다.

## 한계·주의

- 역소문은 소식 슬롯(14개)을 공유하므로 소문 폭주 시 오래된 소식을 밀어낸다.
- 약속 이행 판정은 "기한 창 안에 둘 다 같은 장소" 하나뿐 — 도서관 마을의 '의도적으로 딴 일을
  고르지 않는 한 지켜진다'는 문법을 확률 당김으로 근사했다.
- 관습은 현재 1개(약속 확인)만 창발한다. Project Sid처럼 관습이 여럿 쌓이려면
  파기 외의 누적 축(행사 성공, 역소문 빈도)에도 관습 씨앗을 달아야 한다 — 다음 후보.
