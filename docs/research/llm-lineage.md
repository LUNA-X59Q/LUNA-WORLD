# LLM 계보에서 이식되는 것들 — Smallville · AI Town · AI Village

- **한 줄**: 도서관 마을의 네 영감(전부 LLM 기반)에서 LLM 없이도 살아남는 구조들의 목록.
- **원 조사**: 루나의 도서관 저장소 `docs/research-comparison-2026-07-19.md` — 소스 파일
  단위까지 확인한 심층 비교가 이미 있다. 이 문서는 그 결과를 "LLM 제거" 관점에서 다시 읽은 것.
- **주의**: 프로토타입(`index.html`)의 "연구 대응표" 탭이 이 문서의 축약판을 이미 내장하고 있다.

## 이식 목록 (LLM 없이 살아남는 것)

| 원천 | 구조 | 프로토타입 반영 | 비고 |
|---|---|---|---|
| Smallville (Generative Agents) | **회상 공식**: 최근성·중요도·관련성 가중합으로 기억 검색 | ✅ `retrieve()`가 공식 그대로 | 원본은 `retrieve.py`, 점수화에 LLM 불필요 |
| Smallville | **성찰**: 쌓인 기억을 상위 통찰로 묶기 | ✅ `makeBundle()` — 단, 통찰 문장이 고정 템플릿 | Tracery + 태그 통계로 강화 가능 |
| Smallville | **계획→반응 루프**: 아침 계획, 사건 시 수정 | ✅ `buildPlan()` + `chooseDestination()` | LLM 계획을 규칙 계획으로 치환한 사례 |
| AI Town | **입력 대기열**: 사용자 입력은 틱에서 처리 | ✅ `submitInput()`/`handleInputs()` | 결정론 유지에 필수 |
| AI Town | **단일 엔진 틱**(60tick→1초 스텝) | ✅ 0.05초 고정 스텝 accumulator | |
| AI Village | **혼선과 검증**: 잘못된 믿음의 전파와 교정 | ✅ 소식 confidence + 관측대 검증 | Saving Gemini 사건의 규칙 기반 축소판 |
| Voyager | **기술 서랍**: 완성한 것이 재사용 능력으로 | ✅ `skills` 배열 | |

## 이식 불가능한 것 (정직하게)

- **자유 언어** — 대화의 내용 자체가 창발하는 것은 LLM 고유. 우리는 "무엇이 일어나는가"의
  창발로 승부하고, 언어는 Tracery로 서술만 한다.
- **예상 밖의 목표 설정** — AI Village의 "$2,000 모금하자" 같은 자발적 목표 생성은 불가.
  사건 풀 + 감독 레이어(RimWorld 참고)로 근사한다.
- **자기 서사** — Gemini의 망상과 회복 같은 내면 드라마는 흉내 낼 수 없다. 대신
  Dwarf Fortress식 기억→성격 변화로 "세월에 따라 달라지는 주민"을 만든다.

## 관전 포인트

두 마을이 정확히 반대 극이 된다:

| | 루나의 도서관 AI마을 | LUNA-WORLD |
|---|---|---|
| 창발의 원천 | 말 (LLM) | 구조 (규칙) |
| 관계 | 무수치 — 사건 로그의 해석 | 수치 — 규칙의 입력 |
| 재현성 | 무시드, 되돌릴 수 없음 | 시드, 언제든 재현 |
| 시간 | 턴제, API 비용에 묶임 | 실시간, 무한 배속 |
| 비용 | LLM 호출당 과금 | 0 |

같은 사람이 두 극을 다 지어본다는 것 자체가 이 프로젝트의 연구적 가치다.
